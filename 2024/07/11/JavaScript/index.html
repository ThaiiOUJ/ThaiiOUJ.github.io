  • d = Boolean(d);: 将 d 的值转换成为布尔值,在 JavaScript 中,除了一些特殊的情况（如空字符串 ""、null、undefined、NaN、0 等），所有字符串都会被转换为 true。

  • d = Boolean(d); 的操作：

    • d 最初被赋值为 "0"（一个非空的字符串）。
    • 调用 Boolean(d) 将字符串 "0" 转换为布尔值。因为 "0" 不是空字符串，根据上述规则，它会被转换为 true。
    • 所以，执行完 d = Boolean(d); 后，变量 d 的值将变为 true。

    因此，d = Boolean(d); 并没有冲突，它正确地将字符串 "0" 转换为布尔值 true。

  • console.log(x, typeof x);：输出 x 的值 NaN 和它的类型 "number"。因为 "pizza" 无法转换为有效的数值，所以 Number("pizza") 返回 NaN。

13.Switch语法

switch 语句是一种条件语句，用于根据表达式的值选择性地执行代码块。它可以替代多个 if...else if...else 的结构，使代码更加简洁和可读。

switch 语法结构

  • switch 后面是一个表达式（通常是一个变量或者函数的返回值），它的值会与每个 case 标签后面的值进行比较。
  • 每个 case 标签后面跟着一个期望的值（可以是常量、变量或者表达式），如果 expression 的值等于 case 后面的值，则执行相应的代码块。
  • 如果没有一个 case 的值与 expression 的值相匹配，那么会执行 default 后面的代码块（可选的，可以没有 default）。
  • break; 语句用来跳出 switch 语句。如果没有 break;，则会继续执行下一个 case 中的代码，直到遇到 break; 或者 switch 结束。

注意事项

  • switch 的 case 标签必须是常量，且值的类型必须与 expression 的类型相同。

  • default 分支是可选的，可以在没有匹配的情况下提供一个备用的执行路径。

16.逻辑运算符

在JavaScript中，逻辑运算符用于处理布尔值（true 或 false）的运算。它们常用于控制流程（如条件语句）和复合表达式。主要的逻辑运算符有三个：&&（逻辑与），||（逻辑或），和 !（逻辑非）。

逻辑运算符详解

  1. 逻辑与（&&）

    • 逻辑与运算符会在两个操作数都为 true 时返回 true，否则返回 false。
    • 它具有短路特性：如果第一个操作数为 false，则不会计算第二个操作数，因为整个表达式的结果已经确定为 false。

  2. 逻辑或（||）

    • 逻辑或运算符会在两个操作数中至少有一个为 true 时返回 true，否则返回 false。
    • 它也具有短路特性：如果第一个操作数为 true，则不会计算第二个操作数，因为整个表达式的结果已经确定为 true。

  3. 逻辑非（!）

    • 逻辑非运算符会将操作数的布尔值取反：如果操作数为 true，则返回 false；如果操作数为 false，则返回 true。

17.严格相等运算符和普通的区别

严格相等运算符（=== 和 !==）

严格相等运算符比较两个值是否全等，即值和类型都相同。

  • ===：严格相等运算符。如果两个值和类型都相同，则返回 true；否则返回 false。
  • !==：严格不相等运算符。如果两个值不全等（值不同或类型不同），则返回 true；否则返回 false。

普通相等运算符（== 和 !=）

普通相等运算符在比较时会进行类型转换，即使两个值类型不同，只要在转换后它们的值相等，就返回 true。

  • ==：普通相等运算符。如果两个值在类型转换后相等，则返回 true；否则返回 false。
  • !=：普通不相等运算符。如果两个值在类型转换后不相等，则返回 true；否则返回 false。

示例：

类型转换的详细规则

普通相等运算符会根据以下规则进行类型转换：

  1. 如果一个操作数是布尔值，先将其转换为数字。

  2. 如果一个操作数是字符串，另一个是数字，将字符串转换为数字。

  3. 如果一个操作数是对象，另一个不是，将对象转换为原始值（调用对象的 valueOf 或 toString 方法）。

  4. null 和 undefined 相等。

总结

  • 使用 === 和 !== 时，不会进行类型转换，比较时类型和值都必须相同，结果更为准确和可预测。
  • 使用 == 和 != 时，会进行类型转换，可能导致一些意外的比较结果。因此，通常推荐在JavaScript中使用严格相等运算符和严格不相等运算符来避免类型转换带来的问题。

1. 全局作用域（Global Scope）

2. 函数作用域（Function Scope）

在函数内部声明的变量只能在该函数内部访问。

3. 块级作用域（Block Scope）

使用let和const关键字声明的变量具有块级作用域，只在声明它们的代码块内有效。

4. 词法作用域（Lexical Scope）

内部函数可以访问外部函数中声明的变量，这称为词法作用域或闭包。

使用var关键字声明的变量会被”提升”到其作用域的顶部，但初始化不会被提升。

6. 暂时性死区（Temporal Dead Zone, TDZ）

let和const声明的变量从块的开始到声明之前的区域被称为”暂时性死区”。在此区域内，变量不能被访问。

总结

理解JavaScript的变量作用域对于编写清晰、高效的代码至关重要。记住以下几点：

  1. 全局变量应谨慎使用，以避免命名冲突。
  2. 优先使用let和const来声明变量，以利用块级作用域的优势。
  3. 注意变量提升和暂时性死区的特性，以避免意外行为。
