案例解析

今天是一个简单的用户交互界面/2024/07/19

HTML 结构解释：

DOCTYPE 声明和基本文档结构：

  • <!DOCTYPE html> 声明了 HTML5 文档类型。
  • <html lang="en"> 标签指定了文档的语言为英语。
  • <head> 包含了文档的元数据，如字符编码、视口设置和标题。
  • <meta charset="UTF-8" /> 设置文档的字符编码为 UTF-8。
  • <meta name="viewport" content="width=device-width, initial-scale=1.0" /> 用于确保页面在各种设备上正常显示，视口宽度等于设备宽度，并且初始缩放级别为 1.0。
  • <title>Log_in</title> 设置页面的标题为 “Log_in”。
  • <link rel="stylesheet" href="Log_in.css" /> 引入了外部样式表 “Log_in.css”，用于定义页面的样式。

Body 部分：

  • <h1 id="myH1">Welcome</h1>：显示页面的标题 “Welcome”，使用了 id 属性为 “myH1”，可以通过该 id 在 JavaScript 中找到这个元素。
  • <img id="loginImage" src="https://telegraph-image1-c32.pages.dev/file/b945ad4ea8310f044943c.png" alt="Login image" />：显示一个图像，使用了 id 属性为 “loginImage”，并且设置了 src 属性为一个特定的图像链接，alt 属性为 “Login image”，用于在图像无法显示时提供替代文本。
  • <label for="MyText">Username: </label>：显示一个标签 “Username:”，用于描述输入框。
  • <input id="MyText" />：显示一个文本输入框，使用了 id 属性为 “MyText”，可以通过该 id 在 JavaScript 中获取用户输入的内容。
  • <button id="MySubmit">Submit</button>：显示一个提交按钮，使用了 id 属性为 “MySubmit”，点击该按钮会触发提交动作。
  • <script src="Log_in.js"></script>：引入了外部 JavaScript 文件 “Log_in.js”，用于处理用户交互和逻辑。

css

body 样式解释：

详细解释：

  1. font-family: Verdana, sans-serif;
    • font-family 属性用于设置页面中的字体。首选使用 Verdana 字体，如果用户设备上没有 Verdana 字体，则使用默认的 sans-serif 字体。
  2. font-size: 1em;
    • font-size 属性设置页面中文本的基本大小。1em 表示使用浏览器的默认字体大小。
  3. display: flex;
    • display: flex; 将 body 元素设为弹性布局（Flex 布局）。
  4. flex-direction: column;
    • flex-direction 设置主轴的方向为垂直方向，项目按照列的顺序排列。
  5. align-items: center;
    • align-items 属性定义项目在交叉轴上如何对齐。这里设置为 center，表示在垂直方向上居中对齐所有项目。
  6. justify-content: center;
    • justify-content 属性定义了项目在主轴上的对齐方式。这里设置为 center，表示在水平方向上居中对齐所有项目。
  7. height: 100vh;
    • height 设置为 100vh，即视口高度的 100%。这样设置使得 body 元素的高度始终等于浏览器视口的高度，页面内容能够填满整个屏幕。
  8. margin: 0;
    • margin 设置为 0，消除了 body 元素的默认边距，确保页面内容紧贴边缘。
  9. background-color: #f0f0f0;
    • background-color 属性设置页面的背景颜色为浅灰色（十六进制颜色码 #f0f0f0）。

这些样式的组合使得页面在任何设备上都能以一种简洁、居中的方式展示，并且确保内容能够充分利用视口空间。

- 这些行定义了几个变量和常量来引用 HTML 文档中的一些元素和消息。具体来说：
  - `submitButton` 是提交按钮的引用。
  - `usernameInput` 是用户名输入框的引用。
  - `passwordInput` 是密码输入框的引用。
  - `headingElement` 是页面上的一个标题元素的引用。
  - `imageElement` 是一个图片元素的引用，用于显示登录成功后的图像。
  - `errorMessage` 是一个用来显示错误消息的元素的引用。



- 这里定义了正确的用户名和密码。在实际应用中，这些通常会存储在后端数据库中，而不是硬编码在前端 JavaScript 中。



- `addEventListener` 用于给提交按钮 `MySubmit` 添加一个点击事件的监听器。当用户点击提交按钮时，会执行一个回调函数。

- 如果用户名或密码不正确，就会显示一个错误消息，并将 `errorMessage` 元素的样式设置为 `block`，以便在页面上显示错误信息。
