Vue.js

基本概念

  1. 渐进式框架：Vue.js 是一个渐进式框架，这意味着你可以逐步地将其应用到现有的项目中。你可以只使用 Vue 的视图层，然后根据需要逐步引入其其他功能。
  2. 响应式数据绑定：Vue 的核心特性之一是响应式的数据绑定。它使用双向绑定的方式，当数据变化时，视图会自动更新；同样，当视图发生变化时，数据也会相应更新。
  3. 组件化：Vue.js 允许你将应用拆分成可复用的组件。每个组件包含自己的逻辑（JavaScript）、模板（HTML）和样式（CSS），使得代码更具模块化和可维护性。

安装和引入

初学通过多种方式引入 Vue.js：

  1. 通过 CDN：直接在 HTML 文件中引入 Vue.js 库。

  2. 通过 npm 安装：适用于使用模块化构建工具（如 Webpack 或 Vue CLI）的项目。

  3. 通过下载Vue.js引用

Vue的生命周期:实例生命周期钩子

生命周期钩子的调用顺序总结

  1. beforeCreate: 实例初始化后调用。
  2. created: 实例创建完成，数据和方法已经初始化。
  3. beforeMount: 模板挂载到DOM之前调用。
  4. mounted: 模板挂载到DOM之后调用。
  5. 数据变化时:
    • beforeUpdate: 数据变化导致重新渲染之前调用。
    • updated: 数据变化导致重新渲染之后调用。
  6. beforeDestroy: 实例销毁之前调用。
  7. destroyed: 实例销毁之后调用。

示例

  1. html

  2. js

    1. el: "#app": 指定Vue实例要挂载的DOM元素。
    2. data: 定义了Vue实例的响应式数据。这里有一个 message 属性。
    3. beforeCreate(): 在实例初始化之后，数据观测和事件配置之前被调用。此时，data 和 methods 都还不可用。
    4. created(): 实例创建完成后立即调用。此时可以访问 data 中的数据和 methods 中的方法，但视图还未渲染。
    5. beforeMount(): 在挂载开始之前被调用。此时，模板已经编译完成，但还没有挂载到DOM上。
    6. mounted():
      • 实例被挂载到DOM后调用。
      • 使用 this.$nextTick() 确保整个视图已经渲染完成。这在需要基于渲染后的DOM进行操作时非常有用。
    7. beforeUpdate(): 数据更新时调用，发生在虚拟DOM重新渲染和打补丁之前。
    8. updated():
      • 数据更改导致的虚拟DOM重新渲染和打补丁之后调用。
      • 同样使用 this.$nextTick() 确保视图更新完成后执行某些操作。
    9. methods: 定义了Vue实例的方法。这里有一个 changeMessage 方法，用于更新 message 数据。

Vue的模板语法

Vue.js 使用了基于 HTML 的模板语法，允许开发者声明式地将 DOM 绑定至底层 Vue 实例的数据。所有 Vue.js 的模板都是合法的 HTML，所以能被遵循规范的浏览器和 HTML 解析器解析。

在底层的实现上，Vue 将模板编译成虚拟 DOM 渲染函数。结合响应系统，Vue 能够智能地计算出最少需要重新渲染多少组件，并把 DOM 操作次数减到最少。

如果你熟悉虚拟 DOM 并且偏爱 JavaScript 的原始力量，你也可以不用模板，直接写渲染 (render) 函数，使用可选的 JSX 语法。

    动态绑定: 根据数据动态更新属性或样式。

  2. 属性绑定

    :src="imageUrl" 是一个属性绑定语法（简写形式），它将 Vue 实例中的 imageUrl 数据绑定到 <img> 元素的 src 属性中。最终渲染的 <img> 标签的 src 属性将设置为 https://vuejs.org/images/logo.png。

  4. 多行文本插值

    {{ 1 + 2 }}：直接在模板中计算表达式 1 + 2，结果为 3。

    {{ message.toUpperCase() }}：调用 message 的 toUpperCase 方法，将消息转换为大写字母。

计算属性

计算属性是基于它们的响应式依赖进行缓存的。只在相关响应式依赖发生改变时它们才会重新求值。

基础例子

js代码解析

  1. new Vue({}): 这是创建一个新的Vue实例的语法。Vue实例是一个包含选项的对象，这些选项定义了Vue实例的行为。

  2. el: "#cs": 这是Vue实例的挂载元素。el属性指定了Vue将接管哪个DOM元素。在这个例子中，Vue将查找页面上ID为cs的元素，并将Vue实例挂载到这个元素上。

  3. data() { return { message: "Hello Vue!" }; }: 这是一个返回对象的函数，对象中定义了Vue实例的数据。在这个例子中，有一个名为message的属性，其值为字符串"Hello Vue!"。这个data函数必须返回一个对象，Vue将确保这个对象中定义的属性是响应式的，即当这些属性的值发生变化时，视图会自动更新。

  4. computed: { ... }: 这是定义计算属性的地方。计算属性是基于它们的依赖进行缓存的计算值。只有当依赖发生变化时，计算属性才会重新计算。

  5. reversedMessage() { ... }: 这是computed对象中的一个属性，它定义了一个名为reversedMessage的计算属性。这个属性是一个方法，Vue将自动为这个方法添加this上下文，使其能够访问Vue实例的数据。

  6. return this.message.split("").reverse().join("");: 这是计算属性reversedMessage的实现。它首先通过split("")将message字符串转换为一个字符数组，然后使用reverse()方法将数组中的字符顺序颠倒，最后使用join("")将颠倒后的字符数组重新组合成一个字符串。这个计算属性的值将是"Hello Vue!"字符串颠倒后的"!uVe olleH"。

  7. split(""): 这个方法将一个字符串分割成一个字符数组。split 方法接受一个参数，即分隔符，这里使用的是空字符串 "" 作为分隔符。这意味着原始字符串中的每个字符都将被分割出来，形成一个包含所有单个字符的数组。

  8. reverse(): 这个方法用于数组的元素顺序颠倒。它直接修改原数组，不创建新的数组实例。

计算属性 vs 侦听属性

Vue 提供了一种更通用的方式来观察和响应 Vue 实例上的数据变动：侦听属性。然而，通常更好的做法是使用计算属性而不是命令式的 watch 回调。

js解析

  1. new Vue({}): 创建一个新的Vue实例。
  2. data() { ... }: 这是一个函数，返回Vue实例的数据对象。Vue实例的数据应该是响应式的，Vue内部会使用Object.defineProperty来确保数据的响应性。
  3. return { firstName: "Foo", lastName: "Bar" };: 函数返回一个对象，包含两个属性 firstName 和 lastName，它们的初始值分别是 "Foo" 和 "Bar"。
  4. computed: { ... }: 定义了Vue实例的计算属性。计算属性是基于它们的依赖进行缓存的计算值。只有当依赖项发生变化时，计算属性才会重新计算。
  5. fullName() { ... }: 这是计算属性 fullName 的定义。它是一个函数，Vue将自动为这个函数添加 this 上下文，使其能够访问Vue实例的数据属性。
